LITERAL SOUNDS LABS
WINDOWS INSTALLATION INSTRUCTIONS

These instructions apply to Literal Sounds Labs plugin downloads for Windows.
Install only the formats included in your download.

BEFORE INSTALLING

1. Close your DAW and any standalone version of the plugin.
2. Right-click the downloaded ZIP file and choose Extract All.
3. Do not run the plugin directly from inside the ZIP file.
4. Do not rename the plugin files.

INSTALL THE VST3 PLUGIN

Copy the file ending in .vst3 to:

C:\Program Files\Common Files\VST3\

If Windows asks for administrator permission, approve the copy operation.

The .vst3 item may appear as a single file or as a folder-like plugin bundle.
Copy the complete item without opening or changing its contents.

INSTALL THE STANDALONE APPLICATION

If the download includes a standalone .exe file, place it in a permanent folder,
for example:

C:\Program Files\LiteralSoundsLabs\<PluginName>\

You may create a desktop shortcut after placing it there.

If the download includes a setup installer instead, run the installer and follow
the on-screen instructions.

FIRST LAUNCH

1. Reopen your DAW.
2. Make sure VST3 scanning is enabled in the DAW's plugin preferences.
3. Run a full plugin rescan if the plugin does not appear automatically.
4. Create an audio track for effects or an instrument/MIDI track for synthesizers.
5. Load the plugin from the Literal Sounds Labs or manufacturer list.

Paid plugins can be opened in trial mode where available. To unlock a purchased
copy, enter the Payhip licence key in the plugin's licence section and select
Activate.

IF WINDOWS SMARTSCREEN APPEARS

Only continue if the file was downloaded from the official Literal Sounds Labs
website.

If Windows displays "Windows protected your PC":

1. Select More info.
2. Check that the application name matches the plugin you downloaded.
3. Select Run anyway.

Do not bypass the warning for a file received from an unknown website, email or
file-sharing service.

IF THE PLUGIN DOES NOT APPEAR

1. Confirm that the .vst3 file is located in:

   C:\Program Files\Common Files\VST3\

2. Confirm that your DAW supports 64-bit VST3 plugins.
3. Open the DAW's plugin settings and run a full rescan.
4. Restart the DAW.
5. Make sure the file was fully extracted and is not still inside the ZIP.
6. Make sure the plugin was not placed inside an unnecessary extra folder.

For REAPER, check:

Options > Preferences > Plug-ins > VST

For Ableton Live, check:

Preferences > Plug-Ins > Use VST3 Plug-In System Folders

UPDATING

1. Close the DAW and standalone application.
2. Replace the old .vst3 and .exe files with the newer versions.
3. Reopen the DAW and rescan plugins if required.

Your presets, settings, trial information and licence data are stored separately
from the plugin file, so replacing the plugin normally does not remove them.

UNINSTALLING

Delete the installed plugin from:

C:\Program Files\Common Files\VST3\

Delete the standalone application from the folder where you placed it.

Removing the plugin file does not automatically remove its saved settings or
licence data.
